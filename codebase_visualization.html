ReqBot — Codebase Visualization

A RAG-powered requirements analysis agent. Ingests PDF documents, stores vector embeddings in PostgreSQL, and answers user questions via a Groq LLM.

Data Flows

Ingestion Pipeline Offline / One-Time

1
main.py → ingestion.process_ingestion()

Iterates over every PDF in data/raw/. Assigns a doc-N document ID.

2
pdf_processing.process_page(path)

Reads each PDF page. Converts to Markdown via pymupdf4llm. Optionally describes figures through a VLM (currently commented out).

3
chunking.chunking_page(text)

Splits Markdown text at heading boundaries (#…######). Returns section titles, chunk contents, and sequential IDs.

4
embeddings.create_embedding(chunks)

Encodes each chunk with SentenceTransformer locally. Returns a list of float vectors.

5
INSERT INTO rag_chunks

Stores document_id, chunk_index, content, metadata (JSONB), embedding (vector) via psycopg3.

Query Pipeline Interactive / Real-Time

1
agent.start_agent() — user input loop

Reads user question from stdin. Maintains a rolling message history (system + last 4 turns).

2
retrieval.process_query(query)

Embeds the query string using the same SentenceTransformer. Runs a cosine distance (<=>) search against rag_chunks, returns top 5.

3
Chunk selection (top-1, TODO: rerank)

Currently takes only the single closest chunk. A TODO notes planned re-ranking logic.

4
Groq API — llama-3.3-70b-versatile

Sends system prompt + recent messages with the retrieved chunk as context. Receives a grounded answer.

5
Print response → append to history

Assistant reply is printed to stdout and stored for multi-turn context.

Code Observations & TODOs
TODO — Re-ranking

In agent.py, after retrieval only the top-1 chunk is used. The comment marks intent to add a re-ranking step to select the most relevant chunk from the top-5 candidates.

TODO — VLM Image Description

In pdf_processing.py, the process_vlm() call inside process_page() is commented out. The infrastructure (base64 encoding, HF client, Llama 4 Scout) is in place but disabled.

TODO — Real Embedding API

embeddings.py still contains a fake_embedding() stub with a comment "Add real embedding and API call". The real create_embedding() is implemented but the stub was never removed.

Hardcoded Path

config.py contains an absolute path /home/arda/Desktop/Projects/requirement_project/data/raw. This breaks on any other machine. Should use a relative path or an environment variable.

Hardcoded DB Password

DB_URL in config.py embeds the PostgreSQL password in plain text. This should be moved to an environment variable (e.g. DATABASE_URL in .env).

Extracted Images Path

pdf_processing.py lists "extracted_images" (CWD-relative), which works only when the process is started from the project root. The root-level extracted_images/ dir duplicates data/extracted_images/.

Rolling Context Window

agent.py intelligently slices messages[-4:] plus the system prompt for each API call — preventing unbounded context growth while preserving recent turns.

Generator-Based Ingestion

pdf_processing.process_page() is a Python generator (yield), meaning pages are processed one at a time without loading the entire PDF into memory.

Pure-Python Chunker

chunking.py has zero non-stdlib dependencies. Hierarchical titles (e.g. "Intro / Section / Subsection") provide rich metadata at negligible cost.
